Warning DI002 Roslyn Syntax Analysis

IDataProtectionProvider appears unregistered

Problem: A constructor-injected application dependency does not have a matching registration.

Why it matters: Missing registrations usually surface as startup exceptions or hard-to-trace runtime failures when the container builds the object graph.

Recommended pattern: Every constructor-injected application dependency should be registered in the composition root or a clearly imported module.

Good example:

builder.Services.AddScoped<IPaymentGateway, StripePaymentGateway>();
builder.Services.AddScoped<IOrderWorkflow, OrderWorkflow>();

Bad example:

public sealed class OrderWorkflow
{
    public OrderWorkflow(IPaymentGateway gateway) { }
}
// No IPaymentGateway registration found.

Info SECCONN Heuristic Analysis

Connection string is stored in configuration

Problem: A committed configuration file contains a connection string value.

Why it matters: Source-controlled connection strings can leak credentials, database topology, or privileged connection details across environments.

Recommended pattern: Store production secrets outside committed configuration using environment variables, user secrets, or a managed secret store.

Good example:

"ConnectionStrings": {
  "Default": ""
}
// DOTNET_ConnectionStrings__Default is supplied by deployment configuration.

Bad example:

"ConnectionStrings": {
  "Default": "Server=tcp:prod;User Id=app;Password=P@ssw0rd!"
}

Error SECJWT Heuristic Analysis

Weak JWT configuration value

Problem: JWT issuer, audience, or signing key configuration appears weak.

Why it matters: Weak JWT settings can make issued tokens unverifiable, ambiguous across environments, or vulnerable to replay and signing issues.

Recommended pattern: Use non-empty issuer and audience values plus a high-entropy signing key supplied from secure configuration.

Good example:

"Jwt": {
  "Issuer": "https://identity.example.com",
  "Audience": "dotdet-api",
  "Key": ""
}
// Signing key supplied from a managed secret store.

Bad example:

"Jwt": {
  "Issuer": "",
  "Audience": "",
  "Key": "dev"
}

Microsoft documentation:

  • JWT bearer authentication in ASP.NET Core
Warning SECSECRET Heuristic Analysis

Possible secret stored in configuration

Problem: A configuration key looks sensitive and contains a non-placeholder value.

Why it matters: Secrets committed into configuration files are easy to copy, leak, or reuse in ways that weaken production controls.

Recommended pattern: Keep API keys, tokens, and passwords in managed secret storage rather than source-controlled files.
